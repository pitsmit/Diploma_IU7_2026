# Комбо

порт не фиксированный, а генерируемый
+ ещё проверка сессии
+ хранить можно хэши порта и токена а не их самих
+ требовать sudo для cli команды

Демон запускается:
порт = get_free_port()
токен = generate_token()
bind(127.0.0.1:порт)
сохраняем port и token в ~/.config/myapp/

CLI делает:
port=$(cat ~/.config/myapp/port)
token=$(cat ~/.config/myapp/token)
xdg-open "http://127.0.0.1:$port/?token=$token"

/usr/bin/myapp - тут хранится CLI команда - по факту бинарь который дёргает xdg-open

# Как CLI общается с демоном

* через HTTP(Демон уже веб-сервер)
* Unix socket
* через systemd

Про то, где будет хостится мой интерфейс:
Использовать простой HTTP-сервер (serve, python3 http.server, Nginx локально)

# Файл конфигурации для демона через systemd

[Unit]
Description=MyApp Service
After=network.target

[Service]
ExecStart=/usr/local/bin/myapp daemon
Restart=always
RestartSec=3

# ограничения и защита
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true

[Install]
WantedBy=multi-user.target
